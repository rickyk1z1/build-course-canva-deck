# 整体课程体系约束

## 总体目标

把每一套演示文稿视为同一套“自媒体认知、剪辑思维、视频剪辑技能”课程体系中的一个组成部分，而不是彼此独立的文章。单课内容必须服务于整体知识递进、术语一致性和模块分工。

## 制作前必须建立课程位置

优先在当前工作区查找课程目录、知识地图、既有课件标题、课程体系说明和相邻章节。建立 `curriculum-context.json`：

```json
{
  "system_name": "自媒体与视频剪辑课程体系",
  "module": "当前所属模块",
  "course_role": "本课在整体体系中负责解决的问题",
  "prerequisite_lessons": [],
  "next_lessons": [],
  "shared_terms": {},
  "neighbor_topics": [],
  "excluded_neighbor_topics": []
}
```

如果工作区无法确认模块位置、前置课或相邻课，向用户询问缺失信息。不要凭空创造课程结构。

## 内容优化原则

- 延续既有课程的术语、比喻、难度和表达口径。
- 本课只承担它在体系中的教学任务；前置知识只做必要回顾，不重新完整讲授。
- 对后续课程需要展开的内容，只建立认知接口或一句话预告，不提前抢讲。
- 相邻课程已有完整覆盖的内容，记录交叉引用，不重复堆叠。
- 优化页面衔接和拆页时，同时检查它在整体学习路径中的先后关系；不要借“优化顺序”重排权威源大纲的层级和同级顺序。
- 新增例子应强化当前模块目标，不引入另一模块的操作流程。

## 粗纲扩写约束

粗纲要比当前 `影像基础参数` 定稿更细，但细节只能沿原分支纵向加深，并同时受整体课程分工约束：

1. 新增内容能直接映射到原大纲节点。
2. 新增内容符合本课 `course_role`。
3. 新增内容不属于 `excluded_neighbor_topics`。
4. 如果一个知识点更适合后续课程，将其放入课程交接说明，不进入本课屏显。

例如，本课讲编码与格式时，可以补充定义、关系、体积与兼容性的理解，但不能扩展成剪辑软件按钮操作、代理回连或完整工作流。

## 交付记录

在最终产物目录保留内部文件 `课程体系关联说明.md`，记录：

- 本课在整体体系中的位置；
- 使用了哪些前置知识；
- 为后续课程留下哪些接口；
- 为避免重复而主动排除的相邻主题；
- 本课沿用或新增的统一术语。

该文件不进入 Canva 页面。
